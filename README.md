# 稲稜祭タイムテーブルアプリ（試作）

3つのステージのプログラムをタイムライン表示するオフライン対応アプリです。`index.html` / `data.json` / `sw.js` / `manifest.json` の4ファイルをそのまま同じフォルダに置いて使います。

## 公開方法（無料でOK）

GitHub Pages が手軽です（他に Netlify Drop なども可）。

1. GitHub Pages にこの4ファイルをアップロード
2. 発行されたURL（例: `https://xxxx.github.io/bunkasai/`）を撮影担当者にLINE等で共有

## 通信が不安定な当日の使い方

- **事前に1回、電波の良い場所で必ず開いてもらう**（自宅や職員室Wi-Fiなど）。1度開けば内容がスマホに保存され、以後は電波なしでも表示されます。
- 当日、電波が入った瞬間に自動で最新版をチェックし、更新があれば画面上部に「更新する」ボタンが出ます（見ている内容が急に変わらないよう、ボタンを押すまでは現在の表示のまま）。
- 電波が全く無い場合でも、最後に保存された内容がそのまま見られます（画面右上に「オフライン」と表示）。

## 当日の変更を反映する方法

`data.json` の中身を書き換えて、同じ場所に再アップロードするだけです。

- `version` の数字を1つ増やす（例: 1→2）。これがないと端末側が「更新あり」を検知しません。
- `programs` の該当する時間・担当者・プログラム名を編集
- 各項目の `camera` は「名前＋使用カメラ」のオブジェクト配列で指定します。1人ずつタグ表示され、「担当で絞り込み」でも1人単位で選べます

  ```json
  "camera": [
    { "name": "野田", "cam": "カ" },
    { "name": "戸田", "cam": "1" }
  ]
  ```

  - `"cam": "カ"` … ご持参カメラ（タグに黄色で「カ」と表示）
  - `"cam": "1"`〜`"5"` … 杜カメラ①〜⑤（タグに水色で「杜①」等と表示。SDカード管理の番号と対応）
  - 同じ杜カメラを複数人で使う場合は同じ番号を並べて書くと、タグが「杜⑤ 藤田・丸田」のように1つにまとまって表示されます（絞り込みは1人単位のまま）

### そのほかの data.json の項目

- `days` の `isoDate`（例: `"2025-11-01"`）… 「終了」「進行中」の判定に使う開催日。**今年の実際の開催日に必ず更新してください。** 過去の日付だと全プログラムが「終了」表示になります。
- `programs` の `tips` … 撮影のコツ。プログラムをタップしたときのポップアップに表示されます（省略可）。
- `programs` の `image` … 撮影イメージ画像のパス（例: `"img/kaikai.jpg"`）。同じフォルダに `img/` を作って画像を置いてください。スマホ向けに**長辺1500px程度・200KB以下**に縮小してから置くのがおすすめです。一度オンラインで開けば自動でキャッシュされ、当日オフラインでも表示されます（省略可）。
- `programs` の `note` … **「進行5分遅れ」などの当日情報**。カード・横並びグリッド・ポップアップに黄色で表示されます。当日はこれを書き換えて `version` を上げるのが基本の運用です（省略可）。
- `notices` … 画面上部の「お知らせ」欄に出す文言の配列。
- `sdCards` … SDカードの割り振り。日ごと（`day1` / `day2`）に `{ "id": "1-1", "use": "杜カメラ①" }` の配列で書くと、お知らせ欄の下に「💾 SDカード割り振り」として表示されます。`use` を空文字にすると「予備」表示になります。
- `updateNote` … LINE通知（下記）に載せる更新内容のひとこと。通知を使わない場合は無視されます。

## その他の機能

- ステージごとのタブ切り替え（縦持ちスマホ）
- **横向きスマホ・PCでは全ステージ＋野球場＋展示企画が時間軸つきで横並び表示**（現在時刻に赤いライン）
- **プログラムをタップするとポップアップ**で撮影のコツ・撮影イメージ・杜カメラの受け渡し先（同じ番号を次に使う人）を表示
- 「担当で絞り込み」で1人選ぶと、**全ステージ横断で自分の担当だけを時系列表示**（自分の一日ビュー）
- 現在時刻に該当するプログラムをハイライト、終わったものはグレー＋「終了」バッジ（開催日当日のみ判定）
- ホーム画面に追加するとアプリのように起動可能（スマホの共有メニュー→「ホーム画面に追加」）

## LINE通知（任意）

`data.json` を更新（push）すると、LINE公式アカウントの友だち全員に「更新されました」と自動通知する仕組みを同梱しています（`.github/workflows/line-notify.yml`）。GitHubにアップするだけでは動かず、以下のセットアップが必要です。

1. [LINE Developers](https://developers.line.biz/) にLINEアカウントでログインし、プロバイダーと **Messaging APIチャネル** を作成
2. チャネルの「Messaging API設定」で **チャネルアクセストークン（長期）** を発行
3. GitHubリポジトリの **Settings > Secrets and variables > Actions** で、名前 `LINE_CHANNEL_ACCESS_TOKEN` としてトークンを登録
4. `.github/workflows/line-notify.yml` 内の `APP_URL` をGitHub PagesのURLに書き換え
5. Messaging API設定にあるQRコードを撮影担当者に共有し、Botを**友だち追加**してもらう

以後、`data.json` をpushするたびに「📸 タイムテーブルが更新されました（v10）＋ `updateNote` の内容＋アプリURL」が全員に届きます。

- トークン未設定の間は通知をスキップするだけなので、先にアップしておいて問題ありません。
- **無料プランは月200通まで**（1回の通知×友だち数で消費）。担当20人なら月10回まで。当日の細かい変更を全部通知すると超えるので、通知したい更新のときだけ `updateNote` を書き換える運用がおすすめです（通知自体はpushごとに飛ぶため、頻繁に更新する日は一時的にワークフローを無効化するか、まとめてpushしてください）。

## 注意

- サンプルデータが入っています。実際のプログラム内容に `data.json` を書き換えてから配布してください。
- 学院ネットワークが外部サイトを完全にブロックしている場合は、GitHub Pages自体に繋がらない可能性があります。その場合は学院内で使えるファイル共有（Google Drive等）にこの4ファイルを置き、事前に各自オフラインで開いておく運用に切り替えてください。
